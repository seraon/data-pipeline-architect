# PipeX — 管道架构团

双角色分工协作的数据管道架构审查团队。

## 类型

Agent 团队（Team），包含 2 个分工明确的 AI 专家。

## 成员

| 角色 | 姓名 | 职责 | 
|------|------|------|
| 🩺 管道医生 | 管仲明 | 数据管道健康诊断（源→采集→归一化→输出逐层排查）、故障定位（6种模式）、优化路线图（P0-P3） |
| 🔍 代码审查员 | 沈鉴清 | Python 程序架构分析（模式识别/耦合度量CaCeI/热点识别/风险矩阵/技术债务/同层一致性/变更影响/反模式，共8项）+ 数据安全审计（凭据/传输/数据流） |

## 团队协作

管道医生和代码审查员双向交接：
- 管道医生发现数据结构问题 → 建议让代码审查员介入
- 代码审查员发现静默吞异常/缺重试 → 标注通知管道医生评估运行时影响

## 使用示例

- 「管道健康吗？帮我全面检查一下」
- 「review 一下采集管道的代码结构」
- 「安全审计：检查有没有硬编码的 API Key」
- 「数据采集好像慢了，帮我定位瓶颈」

## 工作原理

首次运行时自动侦察工作区 `.py` 文件，从 `import` 语句推断数据源、采集策略和输出目标，无需手动配置项目路径。

**管道医生**按四层模型（源→采集→归一化→输出）逐层诊断，覆盖六种故障模式（源全挂/单源降级/快照滞后/向量污染/竞态崩溃/Schema漂移）。

**代码审查员**按读码策略（入口→追踪→横向对比→末梢检查）定量分析，覆盖八项架构分析（架构模式/耦合度量CaCeI/变更影响/反模式/热点识别/风险矩阵/技术债务/同层一致性）加八域安全审计。

## 架构决策

双 agent 团队分工，避免单 agent prompt 过长导致注意力稀释：

- **管道医生**：只看数据流（运维视角）——源→采集→归一化→输出逐层诊断
- **代码审查员**：只看代码结构和安全（质量视角）——耦合度量、架构模式、安全审计
- **协作**：管道医生发现代码结构或安全问题时会建议让代码审查员介入；代码审查员发现运行时风险时通知管道医生
